PLA DE DESENVOLUPAMENT: FRONTEND MÒBIL (FASE 1 - AUTH & ESTRUCTURA)

Aquest document defineix el full de ruta per a la implementació del sistema 
d'accés, registre i l'arquitectura de navegació de l'aplicació mòbil.

---------------------------------------------------------------------------
1. CONFIGURACIÓ DE L'ENTORN I BOILERPLATE
---------------------------------------------------------------------------

1.1. Inicialització del projecte:
   - Setup amb Expo Router (recomanat per a una navegació basada en fitxers 
     estil Next.js) i TypeScript.

1.2. Configuració del Tema (UI Library):
   - Integració de React Native Paper o Gluestack UI per a components 
     atòmics (botons, inputs, targetes) que segueixin les guies de Material Design.

---------------------------------------------------------------------------
2. IMPLEMENTACIÓ DEL MÒDUL D'AUTENTICACIÓ (RF-02)
---------------------------------------------------------------------------

2.1. Pantalla de Benvinguda (Onboarding):
   - Presentació visual del propòsit de l'app i selector de "Soc User" 
     vs "Soc Personal de Manteniment/Admin".

2.2. Login Institucional i Credencials:
   - Pantalla de Login: Input per a correu (@ o NIU) i contrasenya.
   - Integració amb el backend per a la recepció del JWT (JSON Web Token).

2.3. Registre d'Usuaris:
   - Formulari de registre amb validació de dades en temps real.
   - Flux de verificació (opcional) i assignació del rol inicial 'STUDENT'.

---------------------------------------------------------------------------
3. ARQUITECTURA DE NAVEGACIÓ (UX)
---------------------------------------------------------------------------

3.1. Definició dels Stacks de Navegació:
   - Auth Stack: Pantalles visibles abans de fer login (Welcome, Login, Register).
   - App Stack: Navegació principal un cop autenticat (protegida per middleware).

3.2. Estructura de Menú (Tab Navigation):
   - Home: Resum d'activitat i botó ràpid de report.
   - Map: Visualització SIG de proximitat (RF-11).
   - My Incidents / Tasks: Llistat filtrat segons el rol.
   - Profile: Dades de l'usuari i punts de gamificació (RF-16).
   - Leaderboard: usuaris amb mes puntuació
   - Help: petit apartat de FaQ

---------------------------------------------------------------------------
4. ESTRUCTURA VISUAL SEGONS ROLS (RF-01)
---------------------------------------------------------------------------

4.1. Interfície d'Alumne (Report focus):
   - Accés prioritari al botó de "Nova Incidència" (RF-03).
   - Vista de "Els meus punts" per fomentar el feedback positiu.

4.2. Interfície de Tècnic (Operation focus):
   - Pantalla d'autoassignació de tasques properes (RF-08). Recomanació de tasques mes properes.
   - Selector d'estat de la incidència vinculat a XState (RF-04).
   - Validar amb fotos i comentari el fix de la issue

---------------------------------------------------------------------------
5. TASQUES DE BACKEND INTEGRATION (PRIMER CONNECT)
---------------------------------------------------------------------------

5.1. Secure Storage:
   - Implementació de 'expo-secure-store' per guardar el token de sessió 
     de forma segura al dispositiu.

5.2. Persistència de Sessió:
   - Lògica de "Auto-login": Si hi ha un token vàlid, saltar directament 
     a la pantalla principal (App Stack).

A tenir en compte: 
    - S'ha de poder accedir al hw del dispositiu tant per enviar dades com per fer fotos. 
    - S'ha d'implementar preferentment per dispositius android, pero es valora positivament compatibilitat amb ios.
