PLA DE DESENVOLUPAMENT: MÒDUL SIG, CLUSTERING I MAPES DE CALOR
(Versió revisada i implementada)

---------------------------------------------------------------------------
FASE 1: TRIGGER POSTGIS + ENDPOINT GEOJSON                    [COMPLETADA]
---------------------------------------------------------------------------

1. Trigger PL/pgSQL (backend/prisma/sql/001_postgis_trigger.sql):
   - Funció sync_report_location() que s'executa BEFORE INSERT OR UPDATE
   - Transforma automàticament latitude + longitude en un punt geography
     (SRID 4326) a la columna 'location'
   - S'ha d'executar manualment a Supabase SQL Editor

2. Endpoints GeoJSON i Heatmap (backend/src/routes/geo.ts):
   - GET /api/geo/geojson → FeatureCollection amb filtres (state, category, days)
   - GET /api/geo/heatmap → Array de punts amb pes configurable
   - Protegits amb authenticate + authorize('ADMIN')

---------------------------------------------------------------------------
FASE 2: MAPA LEAFLET BÀSIC AMB MARKERS                        [COMPLETADA]
---------------------------------------------------------------------------

3. Integració react-leaflet al frontend:
   - Component MapContainer centrat al campus UAB (41.5025, 2.1060)
   - Capa base OpenStreetMap
   - Markers amb color segons estat (OPEN=blau, ASSIGNED=groc, etc.)
   - Popup amb títol, estat, prioritat, categoria i data
   - Clic al marker navega al detall de la incidència

---------------------------------------------------------------------------
FASE 3: CLUSTERING VISUAL AMB LEAFLET.MARKERCLUSTER           [COMPLETADA]
---------------------------------------------------------------------------

4. Plugin leaflet.markercluster:
   - Agrupació automàtica per proximitat amb animacions
   - Dispersió (spiderfy) al fer clic o zoom màxim
   - Component MarkerClusterGroup.tsx amb icones personalitzades

   DECISIÓ: S'ha descartat el clustering SQL (ST_ClusterDBSCAN) ja que
   el volum de dades d'un campus universitari (centenars, no milions)
   no justifica la complexitat addicional. Leaflet.markercluster és
   suficient per a aquest escenari.

---------------------------------------------------------------------------
FASE 4: FILTRES AL MAPA                                        [COMPLETADA]
---------------------------------------------------------------------------

5. Filtres implementats a MapPage.tsx:
   - Per ESTAT: selector amb tots els estats de la màquina XState
   - Per CATEGORIA: selector amb els valors de l'enum Category
   - Per DATA: últims 7, 30 o 90 dies
   - Tots els filtres es passen als endpoints GeoJSON/Heatmap com a
     query params i el backend filtra a nivell de consulta Prisma

---------------------------------------------------------------------------
FASE 5: HEATMAP AMB SELECTOR DE PES                            [COMPLETADA]
---------------------------------------------------------------------------

6. Plugin leaflet.heat amb tres criteris de pes:
   - PRIORITAT: LOW=1, MEDIUM=2, HIGH=3, CRITICAL=4
   - DENSITAT: totes les incidències pesen igual (mostra concentració)
   - ANTIGUITAT: pes proporcional a les setmanes d'edat (1-10)

   L'admin pot alternar entre vista de markers i heatmap amb un toggle,
   i seleccionar el criteri de pes al mode heatmap.
